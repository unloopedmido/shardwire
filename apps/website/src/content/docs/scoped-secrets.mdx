---
title: Scoped Secrets
description: Apply least-privilege permissions per app process.
---

Scoped secrets let each app authenticate with a constrained capability envelope.

## Full-access secret

```ts
server: {
  port: 3001,
  secrets: ['full-access-secret'],
}
```

## Scoped secret

```ts
server: {
  port: 3001,
  secrets: [
    {
      id: 'moderation',
      value: process.env.SHARDWIRE_SECRET_MODERATION!,
      allow: {
        events: ['ready', 'messageCreate', 'interactionCreate'],
        actions: ['deleteMessage', 'banMember', 'deferInteraction', 'editInteractionReply'],
      },
    },
  ],
}
```

Apps should pass matching `secretId`:

```ts
connectBotBridge({
	url: 'wss://bridge.example.com/shardwire',
	secret: process.env.SHARDWIRE_SECRET_MODERATION!,
	secretId: 'moderation',
});
```

## Rotation model

1. Keep old + new secret entries active
2. Roll app env to new secret
3. Remove old entry in next deploy

Keep secret values distinct across scoped entries.

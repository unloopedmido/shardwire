## 1.5.1

This patch release focuses on reliability and docs/operations polish after the docs-site migration.

- fixed reconnect behavior so exponential backoff no longer resets on socket open before auth success
- added stable troubleshooting links in runtime/config errors (`See: https://unloopedmido.github.io/shardwire/errors/#...`)
- added an `Errors` docs page with anchor-based remediation targets for common failures
- updated GitHub Pages config for project-site hosting (`/shardwire` base + HTTPS canonical site URL)
- hardened CI/publish workflows for monorepo migration edge cases and restored docs build checks in CI
- aligned package workspace formatting behavior with workspace-local `.prettierignore` for generated artifacts

## Current line (1.5)

Recent releases focus on stronger diagnostics and safer startup:

- manifest support with filter declaration validation
- strict startup with structured failure reports
- `catalog`, `preflight`, and `explainCapability` workflows
- support for `expectedScope` validation and unused-capability warnings

## Stability direction

Shardwire emphasizes:

- additive API growth over breaking churn
- explicit runtime contracts (`ActionResult`, diagnostics report types)
- operational hardening for real deployments

## Upgrade guidance

When upgrading:

1. re-run strict startup in staging
2. validate scope generation from manifests
3. inspect diagnostics warnings for capability drift
4. confirm deployment transport and proxy invariants

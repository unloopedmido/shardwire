## Transport constraints

- non-loopback app URLs must use `wss://`
- plain `ws://` is accepted only for loopback hosts
- app URL path must match bridge `server.path` (default `/shardwire`)

## Reverse proxy baseline

Terminate TLS at your edge and forward websocket upgrade headers to the bot process.

```nginx
location /shardwire {
  proxy_http_version 1.1;
  proxy_set_header Upgrade $http_upgrade;
  proxy_set_header Connection $connection_upgrade;
  proxy_set_header Host $host;
  proxy_read_timeout 3600s;
  proxy_pass http://127.0.0.1:3001;
}
```

## Runtime limits

Use bot bridge `server` options for operational safety:

- `maxPayloadBytes`
- `maxConnections`
- `maxConcurrentActions`
- `actionQueueTimeoutMs`
- `idempotencyScope`
- `idempotencyTtlMs`

## Process layout

- isolate bot + bridge in a dedicated process
- isolate each app in separate workers/services
- assign scoped secrets per app class

## Graceful shutdown

1. Drain app work intake
2. Close app bridge clients
3. Close bot bridge last

You are **two small programs** away from a working bot: one starts **The Connection** on the bot side, the other connects your **app** and listens for events.

## Choose your path

| Path | Best for | How |
| --- | --- | --- |
| **`npm create shardwire`** | New project outside this repo; uses **`shardwire` from npm** | Run **`npm create shardwire`** (or **`npm create shardwire my-app`**), then follow the printed steps |
| **minimal-bridge** | Smallest Node-only example inside the monorepo | [**minimal-bridge on GitHub**](https://github.com/unloopedmido/shardwire/tree/main/examples/minimal-bridge) |
| **React + Vite** | Browser dashboard / UI with **`@shardwire/react`** | [**react-vite-dashboard**](https://github.com/unloopedmido/shardwire/tree/main/examples/react-vite-dashboard) |
| **npm workspaces** | `packages/bot` + `packages/app` with one root **`.env`** | [**workspace-monorepo**](https://github.com/unloopedmido/shardwire/tree/main/examples/workspace-monorepo) |

<Callout type="info" title="Scaffold from npm">

**`npm create shardwire`** drops a minimal **bot + app** project (same idea as **minimal-bridge**). After install, copy **`.env.example`** to **`.env`**, set **`DISCORD_TOKEN`** and **`SHARDWIRE_SECRET`**, then run **`npm run bot`** and **`npm run app`** in two terminals.

</Callout>

## Prerequisites

- **Node.js 22+** (your app uses the built-in **WebSocket**—see [**Troubleshooting**](/docs/troubleshooting) if something complains).
- A Discord bot **token** and a shared **secret** string you invent (any long random string).

<Steps>

<Step>

Create a `.env` next to the example (or scaffold) with **`DISCORD_TOKEN`** and **`SHARDWIRE_SECRET`** (same secret in both processes). For **Vite** examples, also set **`VITE_SHARDWIRE_SECRET`** to the **same** value — see [**Connection and authentication**](/docs/troubleshooting#connection-and-auth-errors) if the app cannot connect.

</Step>

<Step>

Run **`npm run bot`** — this starts the **bot bridge** (Discord + bridge server).

</Step>

<Step>

Run **`npm run app`** (or **`npm run dev`** for Vite) — this connects your **app** over **The Connection** and subscribes to events.

</Step>

</Steps>

## Common mistakes (quick checks)

- **Secret mismatch** — the string in **`server.secrets`** (bot) must equal **`secret`** (app). For Vite, **`VITE_SHARDWIRE_SECRET`** must match **`SHARDWIRE_SECRET`**. See [**Connection and authentication**](/docs/troubleshooting#connection-and-auth-errors).
- **Wrong URL** — path is usually **`/shardwire`**; loopback **`ws://`** vs remote **`wss://`**. See [**App URL protocol**](/docs/troubleshooting#app-url-protocol) and [**Non-loopback requires wss**](/docs/troubleshooting#app-wss-required).
- **Bot not running first** — start the bot process before the app. See [**Connection and authentication**](/docs/troubleshooting#connection-and-auth-errors).

## The two tiny programs

**Bot side** (holds Discord + exposes the bridge)—under 10 lines of core setup:

```ts
import { createBotBridge } from 'shardwire';

const bridge = createBotBridge({
  token: process.env.DISCORD_TOKEN!,
  intents: ['Guilds', 'GuildMessages', 'GuildMembers', 'MessageContent'],
  server: { port: 3001, secrets: [process.env.SHARDWIRE_SECRET!] },
});
await bridge.ready();
```

**App side** (your logic)—also under 10 lines to connect and subscribe:

```ts
import { connectBotBridge } from 'shardwire/client';

const app = connectBotBridge({
  url: 'ws://127.0.0.1:3001/shardwire',
  secret: process.env.SHARDWIRE_SECRET!,
  appName: 'my-app',
});
app.on('messageCreate', ({ message }) => console.log(message.content));
await app.ready();
```

<Callout type="warn" title="Import path on the app">

In **Vite / browser** bundles, always import from **`shardwire/client`** so the heavy bot runtime is not pulled into the browser. In Node-only scripts you can still use **`shardwire/client`**—it is the same app API.

</Callout>

## What “good” looks like

- The bot terminal prints that the bridge is ready (`bot bridge ready` in official examples).
- The app terminal prints **capabilities** (what events and actions you can use), or the Vite page shows **ready** with capabilities JSON.
- Sending a message your bot can see shows up in the app logs (or UI).

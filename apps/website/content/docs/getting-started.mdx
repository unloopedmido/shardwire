You are **two small scripts** away from a working bot: one starts **The Connection** on the bot side, the other connects your **app** and listens for events.

<Callout type="info" title="Fastest path (today)">

There is no `npx create-shardwire` template yet. The quickest win is the official **minimal-bridge** example in the repo: clone, `npm install`, add a `.env`, run **`npm run bot`** and **`npm run app`** in two terminals.

[**Open minimal-bridge on GitHub →**](https://github.com/unloopedmido/shardwire/tree/main/examples/minimal-bridge)

</Callout>

## Prerequisites

- **Node.js 22+** (your app uses the built-in **WebSocket**—see [**Troubleshooting**](/docs/troubleshooting) if something complains).
- A Discord bot **token** and a shared **secret** string you invent (any long random string).

<Steps>

<Step>

Create a `.env` next to the example with **`DISCORD_TOKEN`** and **`SHARDWIRE_SECRET`** (same secret in both processes).

</Step>

<Step>

Run **`npm run bot`** — this starts the **bot bridge** (Discord + bridge server).

</Step>

<Step>

Run **`npm run app`** — this connects your **app** over **The Connection** and logs incoming messages.

</Step>

</Steps>

## The two tiny programs

**Bot side** (holds Discord + exposes the bridge)—under 10 lines of core setup:

```ts
import { createBotBridge } from 'shardwire';

const bridge = createBotBridge({
  token: process.env.DISCORD_TOKEN!,
  intents: ['Guilds', 'GuildMessages', 'GuildMembers', 'MessageContent'],
  server: { port: 3001, secrets: [process.env.SHARDWIRE_SECRET!] },
});
await bridge.ready();
```

**App side** (your logic)—also under 10 lines to connect and subscribe:

```ts
import { connectBotBridge } from 'shardwire/client';

const app = connectBotBridge({
  url: 'ws://127.0.0.1:3001/shardwire',
  secret: process.env.SHARDWIRE_SECRET!,
  appName: 'my-app',
});
app.on('messageCreate', ({ message }) => console.log(message.content));
await app.ready();
```

<Callout type="warn" title="Import path on the app">

In **Vite / browser** bundles, always import from **`shardwire/client`** so the heavy bot runtime is not pulled into the browser. In Node-only scripts you can still use **`shardwire/client`**—it is the same app API.

</Callout>

## What “good” looks like

- The bot terminal prints that the bridge is ready.
- The app terminal prints **capabilities** (what events and actions you can use).
- Sending a message your bot can see shows up in the app logs.

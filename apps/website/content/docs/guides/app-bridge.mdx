The app bridge is the consumer side of the system.

## Base connection

In **Vite / browser** app bundles, import from `shardwire/client` so the bot runtime (`discord.js`) is not pulled in (see [Troubleshooting](/docs/operations/troubleshooting/#vite-and-zlib-sync)). In Node app scripts, `shardwire` is fine.

```ts
import { connectBotBridge } from 'shardwire/client';

const app = connectBotBridge({
	url: 'ws://127.0.0.1:3001/shardwire',
	secret: process.env.SHARDWIRE_SECRET!,
	appName: 'moderation-worker',
});
```

## Event subscriptions

```ts
app.on('messageCreate', ({ message }) => {
	if (!message.content?.includes('!ping')) return;
	void app.actions.sendMessage({
		channelId: message.channelId,
		content: 'pong',
	});
});
```

## Action calls

Always treat actions as typed request/response calls with explicit failure handling.

```ts
const result = await app.actions.timeoutMember({
	guildId,
	userId,
	durationMs: 60_000,
});

if (!result.ok) {
	console.error(result.error.code, result.error.details);
}
```

## Voice subscriptions and controls

```ts
app.on(
	'voiceStateUpdate',
	({ state }) => {
		if (state.channelId !== 'voice-room-1') return;
		console.log('in target voice room:', state.userId);
	},
	{ voiceChannelId: 'voice-room-1' },
);

const moved = await app.actions.moveMemberVoice({
	guildId,
	userId,
	channelId: 'voice-room-1',
});

if (!moved.ok) {
	console.error(moved.error.code, moved.error.message);
}
```

## React dashboards

Optional **`@shardwire/react`** provides hooks around `connectBotBridge` so dashboard UIs do not need to wire lifecycle by hand. See the README in the repository (`packages/react`) and the [music controller recipe](/docs/guides/recipe-music-controller). The hooks package depends on **`shardwire`** and peers only **`react`**; core **`shardwire`** still does not bundle React.

## What belongs here

- subscriptions
- product workflows
- dashboards and admin tooling
- moderation policies
- metrics/reporting hooks

What does **not** belong here is raw Discord runtime ownership. That stays on the bot side.

# Community pain points: bridging web apps and Discord bots

This note summarizes recurring complaints found in public issue trackers and maps each one to current Shardwire behavior.

| Complaint seen online                                                                                   | Example sources                                                                                                                                  | Shardwire status                                                               | Evidence in Shardwire                                                                                                                                                                                                                     | Practical takeaway                                                                                                                 |
| ------------------------------------------------------------------------------------------------------- | ------------------------------------------------------------------------------------------------------------------------------------------------ | ------------------------------------------------------------------------------ | ----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- | ---------------------------------------------------------------------------------------------------------------------------------- |
| "My app/bot is connected but `interactionCreate` or `messageCreate` never fires."                       | [discord.js#6288](https://github.com/discordjs/discord.js/issues/6288), [discord.js#11489](https://github.com/discordjs/discord.js/issues/11489) | **Mostly fixed (for bridge capability mismatches).**                           | `app.ready()`/`app.on()` throws `BridgeCapabilityError` for disallowed event subscriptions, `error.details.requiredIntents` points to missing intents, and `app.preflight()` reports capability mismatches before subscribing.            | Use `app.preflight({ events, actions })` first, then `app.capabilities()` after connect, then register only allowed subscriptions. |
| "WebSocket auth/transport setup is brittle (`Authentication failed`, `ws://` vs `wss://`, wrong path)." | [discordgo#1257](https://github.com/bwmarrin/discordgo/issues/1257)                                                                              | **Fixed with guardrails.**                                                     | App URLs enforce loopback-only `ws://` and require `wss://` for non-loopback hosts; troubleshooting docs explicitly call out secret/path mismatch and auth failure checks.                                                                | Use exact path `/shardwire`, correct `secret`/`secretId`, and `wss://` outside localhost.                                          |
| "Interaction responses randomly fail (`Unknown interaction` / `This interaction failed`)."              | [discord.js#7005](https://github.com/discordjs/discord.js/issues/7005), [discord.js#9866](https://github.com/discordjs/discord.js/issues/9866)   | **Partially fixed.**                                                           | Built-in `deferInteraction`, `deferUpdateInteraction`, and workflow helpers (`deferThenEditInteractionReply`, `deferUpdateThenEditInteractionReply`) reduce timing mistakes; action calls always return structured `ActionResult` errors. | Defer quickly, then edit/follow up via helper workflows, and always branch on `result.ok`.                                         |
| "Dashboard permissions are too coarse (admin-only)."                                                    | [volvox-bot#124](https://github.com/VolvoxLLC/volvox-bot/issues/124)                                                                             | **Partially fixed (bridge scope), app-level RBAC remains app responsibility.** | Shardwire supports scoped secrets (`allow.events`, `allow.actions`) per app/process, but it does not implement end-user dashboard RBAC.                                                                                                   | Keep Shardwire secret scopes strict, and implement user-role authorization in your web app API/UI.                                 |

## Bottom line

For bridge-specific pain points (event availability, capability mismatch, secret scope, and transport safety), Shardwire already ships first-party guardrails and diagnostics.

For product-layer concerns (per-user dashboard RBAC and business authorization), Shardwire intentionally leaves that to your application.
